# Hugreen SYNC + homewindow.kr 벤치마킹 분석

작성일: 2026-04-28

본 문서는 `https://www.hugreen.kr/sync/creation`의 자동SYNC/직접SYNC와 `https://www.homewindow.kr`를 비교 분석해, 우리 견적 플로우와 DB 보강 포인트를 정리한 문서다.

---

## 질문 요소 전체 목록

### 1. 휴그린 SYNC에서 실제 확인된 질문 요소

#### 공통 시작
- 견적 진입 방식
  - 자동SYNC
  - 직접SYNC

#### 자동SYNC 첫 질문
- 선호 포인트 1개 선택
  - 자동환기
  - 탁 트인 조망
  - 기본성능 업그레이드
  - 고단열 슬림디자인

#### 공통 STEP 1
- 주거형태
  - 아파트
  - 빌라
  - 주택
  - 기타

#### 공통 STEP 2
- 평형
  - 20평대
  - 30평대
  - 40평대
- 확장 여부
  - 확장형
  - 비확장형

#### 공통 STEP 3
- 공간 선택
  - 거실
  - 안방
  - 침실1
  - 침실2
  - 알파룸
  - 주방
  - 다용도실
- 공간은 평형별로 다름
  - 20평대: 거실, 안방, 침실1, 주방, 다용도실
  - 30평대: 거실, 안방, 침실1, 침실2, 알파룸, 주방, 다용도실
  - 40평대: 거실, 안방, 침실1, 침실2, 알파룸, 주방, 다용도실

#### 직접SYNC STEP 4
- 선택한 공간별 제품 선택
  - 자동환기창
  - 유리난간창
  - 스페셜에디션
  - 스마트에디션
  - 프로젝트창
  - 거실분합문
  - 터닝도어

#### 결과/상담 단계
- 예상 총 가격 확인
- 간편 시공 상담 정보 입력
  - 이름
  - 연락처
  - 주소
  - 기타 시공환경 정보

### 2. 휴그린 내부 데이터 기준 제품-공간 매핑

아래는 페이지 내부 `productList`, `combineList`, `areaSizeList` 기준이다.

| 공간 | 확인된 후보 제품 |
|------|------|
| 거실 | 자동환기창, 유리난간창, 스페셜에디션, 스마트에디션 |
| 안방 | 자동환기창, 거실분합문, 유리난간창, 스페셜에디션 |
| 침실1 | 자동환기창, 거실분합문, 스페셜에디션, 스마트에디션 |
| 침실2 | 스페셜에디션, 거실분합문, 스마트에디션 |
| 알파룸 | 유리난간창, 스페셜에디션, 거실분합문, 스마트에디션 |
| 주방 | 프로젝트창, 스페셜에디션, 스마트에디션 |
| 다용도실 | 터닝도어 |

주의:
- `areaSizeList`에는 평형별 표준 규격이 있다.
- `combineList`에는 `manual`, `auto`, `recommend` 3종 조합이 있다.
- 자동SYNC는 선호 포인트(`combine_column`)에 따라 공간별 추천 제품이 바로 매핑된다.

### 3. homewindow.kr 재확인 결과

`homewindow.kr`는 현재 휴그린처럼 단계형 견적 질문 플로우를 전면에 두지 않는다. 확인된 핵심은 아래다.

- 메인 CTA
  - "창호교체 비용이 궁금하신가요?"
  - "무료견적상담, 지금 연락처만 남겨주세요!"
- 실제 수집 항목
  - 연락처(휴대전화번호)
- 개인정보 수집 문구상 용도
  - 창호 교체 상담
  - 견적 안내
  - 시공 일정 협의
  - 고객 문의 응대
- 랜딩 메시지에서 강하게 미는 판매 포인트
  - 15년 무상보증
  - LX 수퍼더블로이
  - TPS 단열간봉
  - 아르곤가스
  - 안전방충망
  - 전국 직영 시공

정리:
- 홈윈도우는 "질문형 견적"보다 "광고형 랜딩 + 즉시 리드 수집" 구조다.
- 따라서 질문 설계 벤치마킹은 휴그린이 중심이고, 전환 방식 벤치마킹은 홈윈도우가 유효하다.

## 자동SYNC vs 직접SYNC 차이점

| 항목 | 자동SYNC | 직접SYNC |
|------|------|------|
| 시작 질문 | 선호 포인트 1개 | 없음 |
| 사용 목적 | 빠른 추천 | 공간별 직접 구성 |
| 사용자 입력량 | 적음 | 많음 |
| 엔진 구조 | 포인트 기반 추천 조합 | 공간별 후보 제품 선택 |
| 결과 방식 | 추천 결과 자동 생성 | 공간별 사용자가 직접 담은 조합 |
| 적합 사용자 | 초보자, 빠른 탐색 | 비교 성향 사용자, 통제 욕구 높은 사용자 |

핵심:
- 자동SYNC는 사실상 `추천형 룰 엔진`이다.
- 직접SYNC는 사실상 `공간별 상품 조합기`다.
- 우리 서비스는 지금 이 둘을 이미 갖고 있으나, 휴그린처럼 "선호 포인트" 입력이 더 앞단에 있어야 선택 피로가 줄어든다.

## 단계별 질문 설계안 (재설계)

### 레벨 1: 10초 가견적

목표:
- 전화 전환용 첫 견적
- 최소 정보만 받기

질문:
1. 주거형태
2. 평형
3. 확장 여부
4. 견적 목적 1개 선택
   - 단열 개선
   - 환기 개선
   - 조망 개선
   - 가성비 우선

출력:
- 대략적인 총액 범위
- LX지인 추천 라인 1개
- 비교 브랜드 대체안 1개

### 레벨 2: 공간별 추천 견적

목표:
- 휴그린 자동SYNC를 우리 구조에 맞게 확장

질문:
1. 레벨 1 질문 유지
2. 교체할 공간 선택
3. 각 공간의 중요 포인트 선택
   - 거실: 조망 / 환기 / 단열 / 디자인
   - 침실: 방음 / 단열 / 가성비
   - 주방: 환기 / 청소편의 / 가성비
   - 다용도실: 출입 / 환기 / 기본성능

출력:
- 공간별 추천 제품
- 공간별 예상가
- 총합

### 레벨 3: 정밀 견적

목표:
- 실제 상담 직전 수준

질문:
1. 레벨 2 질문 유지
2. 각 공간별 실측 또는 규격 범주
   - 소 / 중 / 대 / 특대
   - 또는 가로/세로 직접 입력
3. 설치 환경
   - 저층 / 중층 / 고층
   - 도로변 / 일반 / 조용한 주거지
4. 추가 옵션
   - 로이유리
   - 이중창/시스템창
   - 안전방충망
   - 자동환기
   - 유리난간
5. 브랜드 선호
   - LX지인
   - KCC
   - 실속형
6. 상담 정보
   - 이름
   - 연락처
   - 주소

출력:
- 공간별 상세 견적
- 브랜드 3사 비교
- 추천 이유
- PDF

## 창의적 아이디어 제안

### 1. 휴그린의 "선호 포인트"를 우리식으로 강화

휴그린은 선호 포인트를 1개만 받는다. 우리는 아래처럼 확장할 수 있다.

- 1순위 포인트
- 2순위 포인트
- 절대 포기 못하는 것 1개

예:
- 겨울 난방비
- 도로 소음
- 뷰
- 아이 안전
- 예산

이렇게 하면 추천 근거가 더 설득력 있어진다.

### 2. 홈윈도우식 리드 전환을 레벨 1에 삽입

홈윈도우는 질문보다 전화 전환이 빠르다. 우리도 아래를 넣는 게 좋다.

- 레벨 1 결과 직후
  - "지금 연락처 남기면 3사 비교표 바로 보내드림"
  - "우리집 평형 기준 빠른 상담 예약"

즉:
- 휴그린식 탐색
- 홈윈도우식 전환

둘을 합쳐야 한다.

### 3. 제품명이 아니라 "문제 해결형 질문"으로 바꾸기

제품 질문은 뒤로 보내고, 앞에서는 문제를 받는다.

- 환기가 안 된다
- 난방비가 많이 나온다
- 바깥 소음이 심하다
- 창이 너무 두꺼워 답답하다
- 아이 때문에 안전이 걱정된다

이 입력을 성능 태그 점수와 연결하면 AI 추천이 자연스러워진다.

### 4. 공간별 실측 부담을 줄이는 비정밀 규격 질문

직접 숫자를 묻기 전에 범주형 질문을 먼저 준다.

- 허리 높이
- 키보다 큼
- 천장 가까움
- 베란다 전면형

이후 필요 시만 가로/세로 직접 입력으로 내린다.

### 5. 결과 페이지를 "가격표"가 아니라 "의사결정 보고서"로 변경

휴그린은 추천과 가격 중심이다. 우리는 아래를 같이 보여줄 수 있다.

- 왜 이 제품을 추천하는지
- 대안 제품과 얼마 차이 나는지
- 난방비/소음/환기 기준 기대효과
- 상담 시 확인할 포인트

이게 LX지인 프리미엄 포지션에 더 맞다.

## 결론

핵심 벤치마킹 포인트는 명확하다.

- 질문 구조는 휴그린이 더 강하다.
- 전환 구조는 홈윈도우가 더 공격적이다.
- 우리 현재 DB는 "평형별 표준 구성 + 제품 가격"까지는 충분하다.
- 하지만 "추천 룰", "성능 태그", "옵션 DB", "주거형태 확장"이 부족하다.

따라서 다음 우선순위는 아래가 맞다.

1. 선호 포인트 룰 DB 추가
2. 공간별 후보 제품 매핑 DB 추가
3. 제품 성능 태그 점수화
4. 빌라/주택용 표준 구성 DB 추가
5. 옵션/부자재 DB 추가
