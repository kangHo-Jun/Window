# homewindow.kr 챗봇 심층 벤치마킹 분석 보고서

본 문서는 `homewindow.kr`의 창호 견적 챗봇 시스템을 분석하고, 우리 프로젝트 '지인이'에 적용할 핵심 인사이트를 정리한 보고서입니다.

---

## 1. 항목별 상세 분석

| 분석 항목 | 상세 내용 |
| :--- | :--- |
| **1. 질문 형식** | - **버튼 중심**: 텍스트 입력 최소화, 직관적인 아이콘과 이모지가 포함된 버튼형 인터페이스<br>- **직접 입력**: 정밀한 수치를 원하는 사용자를 위해 각 단계마다 '직접 입력' 옵션 제공 |
| **2. 질문 순서/단계** | - **총 8~10단계**: 건물 유형 → 거주 여부 → 창호 종류(단/이중) → 창 구성(짝 수) → 높이 → 너비 → 수량 → 추가 여부 → 결과 확인 |
| **3. 버튼 텍스트 표현** | - **일상 언어**: "2200mm" 대신 "내 키보다 큰 창", "1200mm" 대신 "내 배꼽 정도 오는 창" 등 신체 부위/공간 명칭 활용<br>- **전문 용어 순화**: 단창/이중창을 "열면 바깥", "창 하나 더" 등으로 풀어서 설명 |
| **4. 디자인 시스템** | - **색상**: 포인트 컬러 Red (#D22727), 배경 White, 텍스트 Black/Gray<br>- **UI 요소**: `rounded-xl` 수준의 큰 라운딩 처리, 버튼 간 충분한 여백 확보 (모바일 오클릭 방지) |
| **5. 레이아웃 구조** | - **데스크톱**: 2컬럼 (좌: 채팅창, 우: 실시간 견적 요약 사이드바)<br>- **모바일**: 1컬럼 (전체 채팅), 우측 하단 플로팅 버튼(FAB)을 통해 견적 요약 레이어 호출 |
| **6. 실시간 업데이트** | - 사용자가 '규격(높이/너비)'을 클릭하는 즉시 우측 사이드바의 가격 데이터가 SSE 또는 상태 업데이트를 통해 즉각 변동됨 |
| **7. 3사 비교 표시** | - LX Z:IN, KCC 홈씨씨, 홈윈도우 자체 브랜드 3종을 수직 리스트 형태로 배치<br>- 브랜드 로고 + 제품 등급 + 부가세 별도 금액 명시 |
| **8. 에러/예외 처리** | - '직접 입력' 시 숫자가 아닌 값 입력 시 입력창 테두리 Red 강조 및 입력 차단<br>- 필수 단계 누락 시 진행 불가하도록 버튼 비활성화 |
| **9. 상담 연결 유도** | - 하단 영역에 상담 전화번호(1661-4830) 상시 노출<br>- 견적 확인 직전 '연락처 입력' 팝업을 통해 자연스럽게 리드(Lead) 수집 |
| **10. 모바일 대응** | - 반응형 웹 설계로 모바일 최적화 레이아웃 제공 (채팅 UI 중심) |

## 2. '지인이' 적용 및 고도화 방안

### ✅ UI/UX 고도화 (Phase 3~4 반영)
1.  **신체 기준 규격 선택 도입**: 
    - 우리도 "키보다 큰 창", "허리 높이 창" 등의 직관적인 버튼을 제공하여 사용자의 측정 부담을 경감.
2.  **실시간 견적 사이드바 구현**: 
    - `QuoteCard` 컴포넌트를 개선하여, 사용자가 대화하는 도중에도 현재까지 입력된 정보 기반의 예상 금액을 실시간으로 노출.
3.  **전문 용어 Tooltip 또는 설명 병기**: 
    - '로이유리', '이중창' 등 전문 용어 등장 시 챗봇이 먼저 "창을 열었을 때 하나 더 있는 게 이중창이에요!" 식의 친절한 설명을 선행.

### ✅ 비즈니스 로직 강화
1.  **3사 비교 테이블 강화**: 
    - 단순 가격 비교를 넘어, LX Z:IN의 프리미엄 가치(단열 성능, 디자인 등)를 경쟁사 대비 부각할 수 있는 지표 추가.
2.  **전환 포인트 최적화**: 
    - 견적 레벨이 올라갈수록(기본→심화) 더 상세한 PDF 레포트나 전문가 유선 상담의 가치를 강조하여 리드 수집율 향상.

## 3. 결론
`homewindow.kr`은 **"비전문가도 1분 만에 견적을 낼 수 있는 직관성"**이 강점입니다. 우리 '지인이'는 여기에 **"AI의 유연한 맥락 이해"**라는 강력한 무기를 더해, 훨씬 더 똑똑하고 친절한 프리미엄 상담 경험을 제공해야 합니다.
