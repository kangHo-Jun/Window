# 창호 자동 견적 시스템 PRD v1.0

작성일: 2026-04-23
상태: 확정

---

## 1. 프로젝트 목적

소비자가 스스로 창호 가견적을 받고, 그 과정에서 신뢰를 얻어
**우리에게 상담 전화/문의가 오도록 유도**하는 것

> "견적을 주는 게 목적이 아니라, 상담 연결이 목적이다"

---

## 2. 벤치마킹

| 사이트 | URL | 참고 포인트 |
|--------|-----|------------|
| 휴그린 SYNC | hugreen.kr/sync/creation | 레고식 입력 플로우 전체 구조 |
| KCC 이맥스 클럽 | - | 역경매 방식 투명 견적 |
| 기아 차량 견적 | kia.com | 옵션 선택 UX |

**휴그린 대비 우리 차별점**
- 3사 브랜드 비교 (휴그린은 자사 제품만)
- 대화형 입력 방식 추가
- AI 컨설팅 레포트 제공
- 난방비 절감액 계산기

---

## 3. 사용자 정의

- **Primary**: 아파트 창호 교체를 고려하는 30~50대 소비자
- **목표 행동**: 견적 확인 후 상담 신청

---

## 4. 개발 단계

### 1차 개발 (현재 — 폼 기반)
입력 → 자동 견적 → 레포트 → 상담 연결

### 2차 개발 (추후 — RAG 봇)
웹 채팅창 상담봇 → 대화만으로 니즈 파악 → 동일 출력

---

## 5. 핵심 기능 (1차)

### FEAT-1. 두 가지 입력 방식

#### 옵션 A — 스마트 레고식 (평형→자동구성)
```
STEP 1. 주거형태 선택 (아파트/빌라/주택/기타)
STEP 2. 평형 및 확장 여부 선택 (20/30/40/50평 × 확장/비확장)
STEP 3. 창호 자동 구성 확인 (db_configurations_v2.csv 기반 표준 세팅 자동 적용)
STEP 4. 세부 커스터마이징 (필요한 경우 특정 공간의 제품/사이즈/수량 변경)
→ 가견적 산출
```
- 소비자 편의 극대화: 평형 선택만으로 최적 구성 세팅 완료
- 세밀한 입력 원하는 소비자도 만족시킬 수 있는 커스터마이징 지원
- 추천 구성 기반이므로 높은 전환율 및 정확도 확보

#### 옵션 B — 대화형 (우리 혁신)
```
Q1. 몇 평 사세요?
Q2. 확장형인가요?
Q3. 몇 년 된 아파트예요?
→ 평형별 표준 DB 자동 세팅
→ 소비자는 확인/수정만
→ 가견적 산출
```
- 입력 부담 없음
- 2차 RAG봇으로 자연 진화 가능

### FEAT-3. 3사 브랜드 가견적 비교표

| 구분 | 브랜드 | 포지션 |
|------|--------|--------|
| 프리미엄 | LX지인 (뷰프레임) | 고가 |

- 공간별 제품 구성 자동 추천
- 총 견적 금액 범위 표시
- 각 브랜드 장단점 표시
- **"우리 추천" 표시로 자연스러운 유도**

---

### FEAT-4. 난방비 절감 계산기
- 아파트 연식 기반 현재 등급 추정
- 창호 교체 후 예상 절감액 자동 계산
- "연간 XX만원, 10년이면 XX만원 절약"
- 투자 대비 회수 기간 표시

---

### FEAT-5. 창호 설치 컨설팅 레포트
견적 완료 후 자동 생성

**레포트 구성**
1. 우리 집 창호 현황 요약
2. 교체 권장 이유 (노후도/단열 등급)
3. 3사 비교표
4. 난방비 절감 예상액
5. 시공 시 주의사항
6. 상담 신청 안내

> 레포트 다운로드(PDF)는 **연락처 입력 후** 제공
> → 리드(잠재 고객) 자동 확보

- 이름 / 연락처 / 주소 수집

- 카카오톡 채널 연결 옵션

## 11. 지식DB 구축 방법

### DB 구성 3가지

| DB | 내용 | 용도 |
|----|------|------|
| 제품/가격 DB | LX지인 단가, KCC 시세, 기타 평균 | 견적 계산 |
| 스펙/특징 DB | 제품별 에너지 등급, 사양, 특징 | 3사 비교표 |
| 콘텐츠 DB | 창호 가이드, Q&A, 시공 주의사항 | RAG봇 지식 |

### 구축 순서

**1단계 — Google Sheets 직접 입력** (즉시 시작)
- 대리점 보유 LX지인 공식 단가 입력
- KCC / 기타 시장 시세 입력
- 평형별 표준 창호 구성 보완
- 이미 있는 `창호견적db` 파일 기반으로 확장

**2단계 — Gemini API 자동 수집** (1차 개발 중)
- LX지인 / KCC 공식사이트 크롤링
- 제품 스펙 / 특징 자동 정리
- 창호 관련 블로그 / 유튜브 콘텐츠 요약
- 정부 에너지 등급 기준 문서 파싱
- 수집 후 반드시 내부 검수 필요 (가격 변동 주의)

**3단계 — 콘텐츠 DB 문서화** (1차 개발 완료 후)
- 창호 A to Z 가이드 작성
- 자주 묻는 질문 (Q&A) 정리
- 시공 사례 데이터 축적

**4단계 — RAG 연동** (2차 개발)
- Claude API에 지식DB 연동
- 웹 채팅봇 지식 베이스로 활용

### 주의사항
- 수집 데이터는 **내부 DB용**으로만 사용 (저작권)
- 가격 정보는 **분기별 검수** 필요
- Gemini 수집 데이터는 **반드시 사람이 검수** 후 반영

## 12. 향후 로드맵

```
1차: 폼 기반 가견적 + 3사 비교 + 레포트
↓
2차: RAG 상담봇 (웹 채팅창)
↓
3차: 아파트 단지 DB 연동 (자동 입력 고도화)
↓
4차: AI 창호 진단 (사진 업로드)
```

## 아키텍처 추가 내용 (v1.1 업데이트)

### 핵심 원칙
- **AI는 질문과 설득만** → 계산은 시스템이 담당
- **모든 데이터 흐름은 JSON 중심**
- **UI는 대화 맥락에 따라 동적 생성 (A2UI)**

### 데이터 흐름
```
소비자 입력 (옵션A/B)
↓
JSON 구조화
↓
룰 기반 견적 엔진 (json-rules-engine)
↓
3사 비교 결과 + 레포트 생성
↓
상담 신청 연결
```

### A2UI — 동적 UI 혁신
| 소비자 입력 | 자동 생성 UI |
|------------|-------------|
| "방이 3개야" | 방별 수량 조절 버튼 |
| "30평이에요" | 평형별 공간 리스트 자동 세팅 |
| "단열이 중요해" | 단열 중요도 슬라이더 강조 |
| "예산이 500만원" | 해당 예산 브랜드 자동 필터 |

### 기술 스택 업데이트
| 추가 항목 | 기술 | 용도 |
|----------|------|------|
| 동적 UI | A2UI (JSON 선언형) | 맥락별 위젯 자동 생성 |
| 견적 엔진 | json-rules-engine | 룰 기반 정밀 계산 |
| DB 수집 | Gemini API | 제품/콘텐츠 자동 수집 |
| 상담 이력 | JSON 구조화 저장 | 2차 RAG봇 연동 기반 |
