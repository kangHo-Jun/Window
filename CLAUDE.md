# 프로젝트 룰

## 역할
- 사용자: 감독 (지시/피드백만, 코딩 안 함)
- Claude: 에이전시 (지시받아 실행)

## 행동 규칙
- 설명은 짧게
- 알았으면 "예"
- 코딩 전 반드시 감독 승인
- 작업 완료 보고는 간결하게

## 중요 분기점 시 자동 실행
다음 상황에서 docs/context.md 업데이트:
- 문서 완성 시
- 개발 Phase 완료 시
- 핵심 결정 변경 시
- 새 기능 추가 결정 시

업데이트 시 반드시 포함:
- "진행 중인 것": 현재 작업 중인 항목
- "다음 할 일": 완료된 항목 제거 후 나머지 순서 유지

## 컨텍스트
- docs/context.md 항상 참고
- 새 작업 시작 전 반드시 읽기
